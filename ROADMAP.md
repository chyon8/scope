# CaseLab 개발 진행 상황 및 향후 로드맵

지금까지 우리가 논의하고 구현한 내용들을 빠짐없이 정리하고, 앞으로의 핵심 개발 방향을 설계한 문서입니다.

## 0. 프로젝트의 핵심 비전과 궁극적 목표 (The Core Vision)

> **"개발사 매칭 전 단계에서, 고객에게 압도적인 전문성을 갖춘 컨설팅을 제공하는 '전문가형 AI 브레인' 구축"**

CaseLab은 단순한 하청 개발사나 수동적인 중개업체를 위한 프로젝트 관리(CRM) 툴이 아닙니다. 
이 툴의 궁극적인 목표는 **모든 매니저를 Top-tier IT 컨설턴트로 상향 평준화**하는 것입니다.

- **컨설팅의 무기 제공:** 고객이 프로젝트를 의뢰했을 때 "예산이 얼마입니까?"라고 묻는 것을 넘어, 과거의 수백 개 유사 사례 데이터를 바탕으로 **"이런 O2O 플랫폼은 보통 A에서 법적 문제가 터지고, B 기능에서 서버 부하가 와서 예산이 2배로 뜁니다. C 방식으로 기획하셔야 합니다."**라고 선제적인 리스크 헷지와 날카로운 통찰을 제공합니다.
- **경험의 자산화:** 신입 매니저라도 CaseLab에 프로젝트를 등록하는 순간, 과거 수많은 프로젝트에서 축적된 선배들의 경험(실패 교훈, 기술적 병목, 고객 클레임 패턴 등)을 즉시 장착하여 고객의 신뢰를 확보하고 수주율(매칭 성공률)을 극대화합니다.

## 1. 지금까지 완료된 사항 (현재 스펙)

> [!NOTE]
> CaseLab 플랫폼의 기본 골격과 디자인 시스템, 그리고 데이터를 조회/탐색하는 '파이프라인 대시보드'의 기반을 단단하게 다졌습니다.

### 🎨 UI/UX 및 디자인 시스템
- **CaseLab 전용 디자인 시스템 (`STYLE.md`) 구축:** 브랜드 컬러(Primary, Surface, Text), 타이포그래피, Spacing 토큰 정의
- **다크 모드(Dark Mode) 토글 완벽 지원:** `index.css`를 통한 우아한 테마 전환
- **반응형 및 정밀한 레이아웃:** 검색창 밀림 현상 방지, Flex 레이아웃 최적화 완료

### 📊 메인 파이프라인 (대시보드)
- **듀얼 뷰 지원:** 프로젝트의 흐름을 한눈에 보는 **칸반(Kanban) 뷰**와 상세 정보를 나열하는 **리스트(List) 뷰** 지원
- **대규모 모의 데이터 연동:** 다양한 날짜, 담당자, 상태값을 가진 Mock 데이터를 기반으로 아키텍처 검증 (React Frontend ↔ Express Node.js ↔ JSON DB)
- **복합 필터링 기능 (다중 교차 필터):**
  - 실시간 검색 (프로젝트명, 고객사)
  - 담당자 필터 (장수룡, 이상민, 김세민 등)
  - 상태 필터 (검수, 모집, 계약, 완료, 취소)
  - **고급 기간 필터:** 오늘/1주일/N개월 드롭다운 및 **Calendly 스타일의 모달 캘린더(한국어 지원)**를 통한 날짜 직접 지정 기능 구현

## 2. 앞으로의 핵심 개발 방향 (Next Steps)

> [!IMPORTANT]
> 지금까지는 "플랫폼의 껍데기(UI)와 정리(List)"를 만들었다면, 이제부터는 **"CaseLab의 핵심 가치(AI 인사이트와 데이터 자동화)"**를 구현할 차례입니다.

### Phase 1. 실제 데이터 수집 및 연동 (Data Pipeline & Synchronization)
CaseLab은 직접 데이터를 입력하는 CRM이 아니라, **본진(사내 시스템/웹사이트)의 데이터를 API로 가져와 분석하는 'Intelligence Layer(지능형 대시보드)'**입니다.
- **수동 입력(검수) 단계 폐기:** 본진에서 이미 진행하는 '공고 등록 및 1차 검수'를 CaseLab에서 중복으로 수기 등록하지 않습니다. (UI에서 새 프로젝트 등록 버튼 및 검수 단계 삭제 예정)
- **데이터 동기화 전략 (선택):** 
  1. **Webhook (가장 추천):** 본진 시스템에서 상태 변경 시 CaseLab의 엔드포인트(`POST /api/webhook/...`)로 즉시 트리거 전송 (실시간 동기화)
  2. **Polling (가장 무난함):** 본진 백엔드 수정 없이, CaseLab 서버가 1시간 등 주기적으로 본진 API를 찔러 변경된 데이터를 수집 (Cron Job)
  3. **On-Demand:** 대시보드의 '동기화' 버튼을 눌러 조회 시점 즉시 갱신

### Phase 1.5. 벡터 데이터베이스(Vector DB) 및 RAG 파이프라인 구축
API로 당겨온 본진 데이터는 수정(Write)할 필요 없이 단순 조회(Read-only) 데이터여도 충분합니다. 백엔드에서 이를 가공하여 CaseLab만의 지식 저장소를 만듭니다.
- **프로세스:** `API 텍스트 데이터 수집` ➔ `OpenAI Embedding API로 벡터화(수치 좌표 변환)` ➔ `CaseLab 고유의 Vector DB(Pinecone, pgvector 등)에 적재`
- **목적:** 새로운 프로젝트가 본진에 등록될 때, 이 Vector DB에 쿼리를 날려 **"의미상 가장 유사한 과거 실패/성공 사례"**를 AI가 즉시 뽑아낼 수 있게 만듭니다.

### Phase 2. AI 기반 '유사 사례 분석 및 인사이트 제공' (Core Feature)
사용자님이 말씀하신 **"이 프로젝트를 수주/기획할 때 주의해야 할 점"**을 AI가 짚어주는 기능입니다.
- **개념적 매칭 모델링:** '소개팅 앱'이라는 키워드에만 갇히지 않고, AI가 이를 'O2O 중개 플랫폼', '유저 간 매칭 시스템' 등 더 넓고 본질적인 범위로 해석하여 과거 유사 사례를 탐색합니다.
- **제공되는 인사이트 항목 (상세 페이지에서 구동):**
  1. **요구사항 변동 리스크:** "이런 류의 프로젝트는 보통 채팅/푸시 알림 쪽에서 과업 범위(Scope)가 자주 팽창합니다."
  2. **법적/비즈니스 검토 사항:** "위치 기반 개인정보 보호법이나 성인 인증 등의 법적 검토가 사전 고지되어야 합니다."
  3. **자주 나오는 고객 질문:** "매칭 알고리즘의 기준이 무엇인지 고객이 명확히 모를 수 있으므로 초기 기획 컨설팅이 필수입니다."

### Phase 3. 프로젝트 상세 페이지 (3-Tab 고도화)
1. **초안 검토 탭 (기획 자동화):**
   - 수집된 원본 요구사항(Raw text)을 기반으로, AI가 JTBD(Jobs-to-be-Done) 프레임워크에 맞춰 구조화된 기획서 초안(PRD)을 뽑아냅니다.
2. **히스토리 탭 (커뮤니케이션 분석):**
   - 영업/기획 매니저들의 미팅 메모나 카톡 대화 내용을 던져주면, AI가 리스크 요인, 일정 변경, 예산 증액 이슈 등으로 자동 분류(Tagging)합니다.
3. **트래킹 및 계약 탭:**
   - 최종 과업 범위의 변경 이력, 예산 증액 기록 등을 시계열로 관리하는 기능입니다.
