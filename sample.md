# 지시사항 (System Prompt)
다음은 프로젝트 의뢰자가 작성한 '처음 내용(Raw Input)'을 전문적이고 구조화된 '정리된 내용(Refined Output)'으로 변환한 예시 데이터 세트입니다. 이 예시들의 패턴(기획서 양식, 어투, 목차 구성 등)을 학습하여, 앞으로 내가 새로운 '처음 내용'을 주면 동일한 형식의 '정리된 내용'으로 변환해 주세요.

# 학습 데이터 (Examples)

## [데이터 1: 아이허브 쿠폰 연동 프로젝트]
### 📥 처음 내용
프로젝트 개요
아이허브(iHerb) 제휴 연동을 기반으로 한 쿠폰(프로모션 할인코드) 관리 기능 추가 및 약알 웹페이지 영문 번역 작업입니다. 아이허브에서 제공하는 프로모션 할인코드를 사용자에게 발급·관리하고, 마이페이지에서 확인·사용할 수 있는 쿠폰 시스템 구축이 핵심입니다. 병행하여 기존 국문 서비스(www.yak-al.com)의 영문 버전(www.us.yak-al.com) 번역 및 로컬라이징 작업을 포함합니다.

지원 플랫폼
PC 웹사이트, 모바일 웹사이트 (반응형). 모바일 애플리케이션은 현 단계에서는 제외하되, 향후 확장 가능한 구조 권장.
솔루션 사용 유무
자체 구축 (기존 약알 플랫폼에 기능 추가 형태). 신규 솔루션 도입이 아닌 현행 서비스와의 연동·통합이 전제입니다.
사용자 메뉴 구성
성분 비교, 약알 Pick(약사 큐레이션), 칼럼, 마이페이지(쿠폰 보관함 포함), 프로모션/이벤트 페이지.
사용자 주요 기능
- 아이허브 프로모션 할인코드 기반 쿠폰 발급·보관·사용
- 마이페이지 쿠폰 관리(보유/사용완료/만료 구분)
- 쿠폰 발급 프로모션 페이지(배너, 알럿/팝업 등 유입 UI)
- 소셜 로그인(기존 연동 유지)
- 웹페이지 영문 번역 및 로컬라이징(us.yak-al.com)

관리자 주요 기능
- 유저 웹: 마이페이지 쿠폰 관리, 쿠폰 발급 프로모션 페이지(배너·알럿 등)
- 관리자 웹: 유저별 쿠폰 관리, 쿠폰 관리(생성·수정·유효기간·발급조건), 프로모션 관리
- 서버: 유저–관리자 웹 API 개발, 이메일·푸시 알림 발송

기능 및 디자인 참고 사이트
기능은 아이허브의 쿠폰/프로모션 구조를 참고하며, 디자인은 기존 약알 서비스의 톤앤매너를 유지합니다.
지원 시 참고 사항
풀스택 개발자 또는 개발사(프리랜서 포함) 구인
본 프로젝트는 아이허브 제휴(impact.com / Partnerize) 연동 경험이 있거나, 커머스 쿠폰 시스템 구축 경험이 있는 분을 우대합니다.
영문 번역·로컬라이징 작업이 병행되므로 다국어(i18n) 처리 경험이 있으면 유리합니다.

## [데이터 2: TM CRM 앱 개발]
### 📥 처음 내용
프로젝트 개요 :TM CRM 앱 개발
200~250명 규모의 TM 마케터 부서에서 사용할 TM CRM 앱이 필요합니다.
개인 모바일로 통화 하는 내용을 AI가 녹취/텍스트 변환 하는 기능
기본 모바일 CRM 기능: 일정 관리, 고객 관리 기타 등등
기존 사용중인 CRM 여부 : X
사용자 수 : 약 200~250명 (사용 예정인 임직원의 대략적인 규모를 알려주세요.)
대표 메뉴 : 고객 정보 관리, 영업 관리, 영업 분석, 고객 서비스 및 지원, 통계 및 보고, 시스템 관리, 녹취, 텍스트 변환 등
지원 플랫폼 : 모바일 전용 어플리케이션
연동 필요한 외부 시스템 : 추후 웹 CRM과의 연동으로 상호 작용까지도 구현 할 계획입니다.

---
### 📤 정리된 내용
[프로젝트 개요]
- 200~250명 규모의 TM 마케터 부서에서 사용할 모바일 기반의 TM CRM 앱 및 관리자용 웹 시스템 신규 구축

[프로젝트 배경 및 목표]
현재 마케터들이 유선 전화가 아닌 개인 휴대전화를 사용하여 영업 통화를 진행하고 있으며, 통화 종료 후 엑셀을 이용하여 수기로 내역을 관리하고 있어 데이터의 정확성과 업무 효율성이 떨어지는 상황입니다. 마케터들의 연령대가 다소 높은 편이므로 직관적이고 쉬운 모바일 앱 UI가 필수적입니다. 본 프로젝트의 목표는 모바일 앱 내에서 인터넷 전화 방식을 통해 통화를 진행하여 통화 내용을 녹음하고, AI를 통해 텍스트 변환(STT) 및 내용 요약을 자동화하는 것입니다. 또한, 관리자가 웹 페이지에서 변환된 데이터와 영업 상태를 통합적으로 모니터링할 수 있는 신규 CRM 시스템을 구축하고자 합니다. 기존 데이터 마이그레이션 없이 신규 데이터부터 축적할 예정입니다.

[과업 범위]
1. 수행 범위
- 상세 기획: 요구사항 정의, 화면 설계, 기능 명세, VoIP 연동 기획, AI API 연동 기획
- UI/UX 디자인: 고령의 마케터가 쉽게 사용할 수 있는 직관적인 모바일 앱 및 관리자용 웹 디자인
- 프런트엔드/Client 개발: iOS 및 Android 모바일 애플리케이션 개발, 관리자 웹 페이지 프론트엔드 개발
- 백엔드 개발: CRM 비즈니스 로직, 인터넷 전화(VoIP) 인프라 또는 API 연동, AI STT 및 요약 API 연동 서버 개발
- 서버/DB/인프라 구성: 다수의 동시 통화 녹음 파일 및 텍스트 데이터를 처리하고 저장하기 위한 클라우드 환경 구축

2. 상세 기능 요구 사항
2-1. 모바일 앱 내 통화 및 녹취 기능 (핵심)
- 인터넷 전화(VoIP) 발신: 일반 이동통신망이 아닌 데이터 통신 기반의 자체 통화 연결 기능 구현
- 통화 녹음 및 전송: 통화 시작 시 오디오 데이터를 서버로 자동 녹음 및 실시간 또는 종료 직후 전송

2-2. AI 텍스트 변환 및 요약 처리
- 음성 텍스트 변환(STT): 수신된 녹음 파일을 텍스트로 자동 변환하는 AI API 연동
- 통화 내용 요약: 변환된 전체 텍스트를 기반으로 핵심 내용을 요약하는 AI API 연동
- 데이터 매핑: 통화 세션 정보, 녹음 오디오 파일, 전체 텍스트 변환본, AI 요약본 정보를 수신하여 해당 고객 데이터와 하나의 항목으로 연결하여 DB에 저장

2-3. 모바일 CRM 앱 기능
- 고객 정보 관리: 신규 고객 정보 등록, 기존 고객 정보 조회, 고객 정보 수정
- 일정 관리: 고객별 재연락 및 미팅 일정 등록, 알림 기능 제공
- 영업 관리: 통화 후 영업 진행 단계별 상태 업데이트, 과거 영업 이력 조회
- 통계 및 보고 조회: 마케터 본인의 개인 통화 건수, 영업 달성률 통계 확인

2-4. 관리자용 웹 CRM 시스템
- 권한 관리: 최고 관리자, 중간 관리자, 일반 마케터 등급을 분리하여 메뉴 접근 권한 부여 및 제어
- 통화 이력 및 녹취 열람: 마케터가 진행한 통화의 녹음 파일 청취, 전체 변환 텍스트 조회, AI 요약 내용 열람
- 고객 서비스 및 지원 모니터링: 전체 고객 데이터 조회, 마케터별 영업 파이프라인 진행 상태 확인, 지원 이력 통합 모니터링
- 시스템 관리: 전사 공지사항 등록, 사용자(마케터) 계정 생성 및 정지, 기초 환경 설정
- 영업 분석: 부서별 전체 통계, 개인별 실적 데이터 추출 및 시각화

3. 비기능적 요구사항
3-1. 성능/규격: 250명의 사용자가 동시 다발적으로 통화를 진행하고 음성 파일을 업로드할 때 서버 지연이나 음성 데이터 누락이 발생하지 않도록 안정적인 동시성 처리 아키텍처를 구성해야 합니다.
3-2. 보안/인증: 민감한 개인정보와 음성 녹음 데이터를 취급하므로 개인정보보호법에 준거하여 데이터베이스를 암호화하고, 클라이언트와 서버 간의 모든 통신 구간에 SSL 암호화를 적용해야 합니다.

[기술/제조 스택]

## [데이터 5: 악성 게시물 증거 수집 SaaS]
### 📥 처음 내용
의뢰 배경, 달성하려는 목적현재 온라인 악성 게시물의 증거를 신속하고 객관적으로 확보하는 웹 기반 SaaS 서비스(Enclow)를 개발하고 있습니다.현재 온라인에서 명예훼손, 허위사실 유포, 악성 리뷰 등의 게시물이 발생하면 대부분 사용자가 직접 캡처하고 URL을 저장해야 하며, 게시물이 삭제되거나 수정되면 증거 확보가 어려운 문제가 있습니다.저희는 이러한 문제를 해결하기 위해 URL만 입력하면 증거를 자동으로 수집하고 하나의 증거 패키지(PDF)로 생성하는 MVP를 만들고자 합니다.
개발 목표사용자가 문제 게시글 URL을 입력하면해당 페이지 접속화면 캡처HTML 또는 본문 저장수집 시각 기록SHA-256 해시 생성을 자동 수행하고,최종적으로 하나의 증거 리포트(PDF) 를 생성하는 웹 서비스를 개발하고 싶습니다.추가적으로 가능하다면브랜드명이나 키워드를 입력했을 때 공개 검색 결과를 수집하거나,공개 게시판(로그인이 필요 없는 게시판)의 게시글 URL을 일부 수집하는 기능도 함께 구현하고 싶습니다.이번 MVP에서 중요하게 생각하는 부분저희가 가장 중요하게 생각하는 것은'증거의 무결성을 확보하는 것' 입니다.복잡한 AI 기능이나 대규모 크롤링보다URL 기반으로 증거를 안전하게 보존하고 관리하는 기능을 우선 구현하고자 합니다.
지원 시 참고 사항 :파트너스가 충족해야 하는 조건이나 우대 사항 등을 알려주세요.[예시] 유사한 프로젝트 경험이 있는 업체를 찾고 있으며 디자인에 역량이 있는 업체를 선호합니다.
